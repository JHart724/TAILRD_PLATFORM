Logged as tech debt #22 (MEDIUM). All subsequent DNS work for SES verification must be done at Wix DNS (manual, via Wix UI), not Route 53.

## 8. Rollback

If SES setup needs to be reversed entirely:

1. Delete domain identity: `aws sesv2 delete-email-identity --email-identity tailrd-heart.com --region us-east-1` (irreversibly deletes DKIM tokens — re-creating regenerates new tokens, requiring fresh DNS publication).
2. Delete sandbox recipient: `aws sesv2 delete-email-identity --email-identity jhart@hartconnltd.com --region us-east-1`.
3. Reverse Route 53 batch: change-batch with `Action: DELETE` on the same 5 records published in §3.2.
4. Revert Wix DNS: restore old SPF (`v=spf1 include:_spf.google.com ~all`) and old DMARC (`v=DMARC1; p=none;`); delete the 3 DKIM CNAMEs.
5. Production access request `DENIED` state can be left in place (no negative effect).

No code changes have been made yet — emailService.ts dynamic-import fallback continues to log to console rather than send. Rollback at any step before code-wiring is fully reversible.
